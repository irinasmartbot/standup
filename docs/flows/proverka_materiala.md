# Воронка «StandUp Проверка материала»

## 1. Источник данных

Мероприятия загружаются из Google Таблицы (лист «Афиша бесплатников») по публичной CSV-ссылке.  
Бот читает таблицу при каждом запросе — данные всегда актуальны.

> **Следующий шаг по плану:** мероприятия переедут в базу данных PostgreSQL.  
> Google Таблица останется как источник синхронизации до появления Админ-панели.

### Структура таблицы

| Столбец | Содержимое |
|---|---|
| A | Порядковый номер записи |
| B | Дата мероприятия (например: 06.06.2026) |
| C | День недели (например: суббота) |
| D | Время начала (например: 17:00) |
| E | Площадка и полный адрес |
| F | Описание мероприятия |
| G | Ссылка на картинку (афишу) |
| H, I | rowbr — внутренние номера (не используются) |
| J | Дополнительные места от бара (например: -40 = +40 к базе) |
| K | Локация — короткое название (например: Escobar) |
| M | Количество уже забронированных мест |
| N | Билет фикс — M + J = занято мест итого |
| Q | Статус: «Актуально» или «Прошло» |

### Логика расчёта мест

- Базовое число мест: **60**
- Максимум мест = 60 + |значение столбца J| (например: J = -40 → максимум 100 мест)
- Бот суммирует все активные брони и не даёт превысить лимит
- Показываются только мероприятия со статусом **«Актуально»** и датой не раньше сегодня

## 2. Схема воронки

```
Главное меню
    ↓
Забронировать места → Выбор формата
    ↓
StandUp Проверка материала
    ↓
Выбор по дате / Выбор по площадке
    ↓
Карточка мероприятия
    ↓
Забронировать билеты
    ↓
Шаг 1: Имя → Шаг 2: Телефон → Шаг 3: Количество гостей
    ↓
Подтверждение брони (запись в БД)
    ↓
Напоминания → Получить билет (подтверждение) или Аннулирование
```

---

## 3. Выбор мероприятия

### Выбор по дате
- Фото + текст «Выбирай дату:» + inline-кнопки с датами
- Даты из столбца B, только статус «Актуально», без прошедших
- Отсортированы по возрастанию, дубли удалены
- Дополнительная кнопка: «📍 Выбор по площадке»

### Выбор по площадке
- Фото + текст «Выбирай локацию:» + inline-кнопки с площадками
- Площадки из столбца K, только актуальные, без повторов
- После выбора площадки — список мероприятий этой площадки:
  ```
  📅 06 июня (сб) 17:00
  📅 07 июня (вс) 19:00
  ```
- Дополнительная кнопка: «📅 Выбор по дате»

### Карточка мероприятия
Если на дату одно мероприятие → сразу карточка.  
Если два и более → сначала выбор по времени/площадке, затем карточка.

Карточка содержит:
- Фото из столбца G
- Дата + день недели
- Время
- Площадка и адрес
- Описание

Кнопки карточки:

| Кнопка | Действие |
|---|---|
| 🎟 Забронировать билеты | Запускает воронку бронирования |
| 📋 Правила бронирования | Показывает 4 правила |
| ◀️ Назад | Возврат к выбору даты/площадки |

---

## 4. Воронка бронирования

### Защита от повторного бронирования
Один Telegram-аккаунт = одна активная бронь на одно мероприятие.  
На разные даты/мероприятия — бронировать можно.

При повторной попытке бот показывает:
```
⚠️ ВНИМАНИЕ, мы уже внесли Вас в списки гостей:
Дата / Время / Локация / Количество гостей
Вы не можете забронировать повторный билет на данное мероприятие
```
Кнопки: Отменить бронь / Изменить дату / Задать вопрос менеджеру

### Шаг 1 — Имя
Бот подтягивает имя из Telegram-профиля и предлагает подтвердить.
- «Все верно 👌» → переход к шагу 2
- «Изменить» → ввод имени вручную → шаг 2

### Шаг 2 — Телефон
Бот предлагает поделиться контактом или ввести вручную.
- Кнопка «📱 Поделиться номером» → Telegram передаёт номер автоматически
- Кнопка «✏️ Ввести вручную» → ввод текстом

### Шаг 3 — Количество гостей
«[Имя], напишите пожалуйста цифрой, на какое количество человек бронируете?»  
Ограничение: **максимум 4 человека** на одну бронь.
- Больше 4 → предупреждение, повторный запрос
- Не цифра → предупреждение, повторный запрос
- Превышение лимита мест → предупреждение с доступным остатком

## 5. Подтверждение брони

После заполнения данных бот показывает:
```
Отлично! Мы внесли Вас в списки гостей:

Дата: 06 июня (суббота)
Время: 17:00
Локация: Escobar, ул. ...
Количество гостей: 2 чел.
```

Бронь записывается в БД со статусом **«booked»**.

### Если бронь в день шоу или за день до него
Кнопка «🎟 Получить билет» появляется **сразу** (не ждём напоминания).  
Добавляется предупреждение: «ОБЯЗАТЕЛЬНО подтвердите бронь, нажав на кнопку «Получить билет»»

### Стандартный случай (более 1 дня до шоу)
Кнопки: Отменить бронь / Изменить дату / Изменить количество гостей / Менеджер / Канал

Текст: «За сутки до мероприятия Вам придёт напоминание с кнопкой «Получить билет». Обязательно нажмите её, чтобы подтвердить бронь.»

---

## 6. Управление бронью

### Отменить бронь
1. Запрос подтверждения: «Для подтверждения отмены брони нажмите кнопку ниже» + «Подтверждаю»
2. Статус в БД → **«cancelled»**
3. Сообщение: «Хорошо, спасибо, что предупредили 😊 Ждём Вас на других мероприятиях...»

### Изменить дату
1. Запрос подтверждения
2. Предыдущая бронь → **«cancelled»**
3. Возврат к экрану выбора дат — пользователь проходит воронку заново

### Изменить количество гостей
1. Запрос подтверждения
2. Новый ввод количества (те же правила: 1–4, проверка лимита)
3. Обновление в БД
4. Сообщение: «Спасибо, количество гостей изменено на [N]. Будем ждать Вас [дата] 👍»

## 7. Система напоминаний

### Напоминание 1 — за сутки (в 14:00)
Отправляется накануне мероприятия в 14:00 по московскому времени.  
Содержит: дату, время, адрес, количество гостей.  
Кнопки: 🎟 Получить билет / Отменить бронь / Изменить дату / Изменить количество гостей

### Напоминание 2 — в день шоу (в 10:00)
Отправляется в 10:00 в день мероприятия, если билет ещё не получен.  
Текст: «[Имя], мне необходимо подтвердить либо отменить Вашу бронь на [дата] в [время]»

### Если бронь сделана в день шоу
- Кнопка «Получить билет» — сразу после бронирования
- Напоминание — через 1 час после бронирования
- Если до шоу менее 1 часа — напоминание через 15–20 минут

### Аннулирование (статус «annulled»)
Если пользователь не нажал «Получить билет»:

| Когда сделана бронь | Когда аннулируется |
|---|---|
| За 3+ часа до шоу | За 2 часа до начала |
| Менее чем за 3 часа | Через 30 минут после последнего напоминания |

Текст: «Ваша бронь аннулирована, ждём Вас на других мероприятиях 😊»  
Кнопка: «Перейти в главное меню»

---

## 8. Получение билета

После нажатия «🎟 Получить билет»:
1. Генерируется изображение на основе шаблона (библиотека Pillow)
2. Статус брони → **«confirmed»**

### Что отображается на билете
- Логотип Moscow StandUp Show
- Имя пользователя
- Дата (06.06.2026)
- Время (17:00)
- Локация (короткое название + адрес)
- Количество гостей (2 гостя)

После билета — второе сообщение с напоминанием об отмене и контактами менеджера.

## 9. База данных — таблица bookings

| Поле | Тип | Описание |
|---|---|---|
| id | INTEGER | Уникальный номер записи |
| telegram_id | INTEGER | ID пользователя в Telegram |
| username | TEXT | Никнейм в Telegram |
| name | TEXT | Имя (из профиля или введённое вручную) |
| phone | TEXT | Телефон |
| event_date | TEXT | Дата мероприятия (ДД.ММ.ГГГГ) |
| event_time | TEXT | Время мероприятия |
| event_address | TEXT | Полный адрес |
| event_location | TEXT | Короткое название площадки |
| guests | INTEGER | Количество гостей (1–4) |
| status | TEXT | booked / confirmed / cancelled / annulled |
| created_at | TEXT | Дата и время создания брони |
| reminder_24h_sent | INTEGER | Отправлено ли напоминание за сутки (0/1) |
| reminder_day_sent | INTEGER | Отправлено ли напоминание в день шоу (0/1) |
| annulled_at | TEXT | Дата и время аннулирования |

### Статусы брони

| Статус | Описание |
|---|---|
| `booked` | Бронь создана, билет не получен |
| `confirmed` | Пользователь нажал «Получить билет» |
| `cancelled` | Пользователь отменил сам |
| `annulled` | Аннулировано системой (не подтвердил вовремя) |

---

## 10. Защита от ввода текста

На всех этапах ожидания кнопки — текстовый ввод игнорируется.  
Бот отвечает: «Пожалуйста, выбери вариант из кнопок ниже 👇»

**Исключения** — этапы где ввод текста обязателен:
- Имя (при ручном вводе)
- Телефон (при ручном вводе)
- Количество гостей
